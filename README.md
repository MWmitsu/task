# 📌 やること（Web版）— iPad・スマホ・どのブラウザでも

Windows常駐ウィジェットと**同じ感覚で使えるタスク管理**を、**iPad（Safari）やスマホ、どのPCのブラウザ**でも動くようにしたWeb版です。インストール不要・追加費用なし・データは端末の中だけ（ブラウザの保存領域）＋お好みでクラウド同期。

**公開URL： https://mwmitsu.github.io/task/ （PWA：ホーム画面に追加するとアプリのように・オフラインでも起動）**

`index.html` 単体でも動きますが、ホーム画面アプリ化・オフライン起動には同じフォルダの `manifest.webmanifest` / `sw.js` / `icon-180.png` / `icon-512.png` が必要です（公開URLにはすべて含まれています）。

## iPadで使う（おすすめ：公開URLをホーム画面に追加）
1. Safariで **https://mwmitsu.github.io/task/** を開く
2. **共有ボタン → 「ホーム画面に追加」**
3. ホーム画面のアイコン（白いチェックマーク）から、**アプリのように全画面**で起動。Service Workerにより**オフラインでも開けます**

> ファイルとして配る場合（`index.html` をOneDrive/AirDrop等で渡す）も使えますが、`file://` ではGoogleログイン・オフラインキャッシュは無効です。データはその端末内に保存されます。履歴消去で消えるので、大事なときは「⇅ → JSON書き出し」でバックアップを。

## できること（Windows版の主要機能を網羅）
- タスク追加（改行で複数まとめて・`- ・ 1.` の記号は自動除去）
- 「**明日 ○○**」「**金曜 ○○**」で締切を自動セット
- 分類（すぐ／通常／いつでも）・フィルタタブ（すべて／今日／すぐ／いつでも）
- 締切が今日・期限切れは「今日」タブに集約（Windows版と同じ挙動）
- 📍優先度（低／中／高）・🏷タグ（タップでそのタグを検索）・📐見積もり・📝備考
- 🔁繰り返し（毎日／平日／毎週）・☑サブタスク（進捗バッジ付き）・締切は**カレンダー入力**でも指定可
- **🔁 繰り返しの自動再生成**：繰り返しタスクを完了すると、次回の締切に自動で繰り越し（連続記録にもカウント）
- **⏰ リマインダー通知**：編集でアラーム時刻を設定すると、その時刻にブラウザ通知＋トースト（初回に通知の許可を確認）
- **⏱ 作業時間タイマー**：編集で ▶開始／■停止。経過時間を行にバッジ表示（アプリを閉じていた時間は数えません）
- 完了（取り消し線）・今日の完了数＆連続日数（🔥）
- 検索・一時メモ・並べ替え・複製・完了を隠す/消す
- **👆 スワイプ操作**：右スワイプで完了／左スワイプで削除（削除は数秒間「タップで取り消し」可）
- **🖱 ドラッグ並べ替え**：手動並び順のとき、行の●（左の点）をつかんで上下に移動
- **↕ 並び順**：右上「↕」で 手動／締切が近い順／優先度が高い順／見積もりが大きい順 を切替
- **🌙 テーマ**：右上「🌙/☀️」でライト／ダークを切替（端末に記憶）

## クラウド同期（複数端末で自動・麻雀アプリと同じ方式）
右上「⇅」→「**🔑 Googleでログイン**」すると、**iPad・PCのブラウザなど複数端末で自動同期**されます（麻雀スコアアプリと同じ Firebase 方式）。
- ログイン中は、編集すると自動でクラウドへ保存され、他端末にもすぐ反映（`onSnapshot`）
- 端末とクラウドの両方にデータがある初回は「どちらを使うか」を確認
- ログイン直後の**突き合わせ中はローカル→クラウドの送信を保留**し、空データでの上書きを防止（認証世代ガード付き）
- **編集シートを開いている間はリモートの反映を保留**（入力中の取りこぼし防止）
- 同期に失敗しても次の変更時に自動で再送（失敗はトーストで通知）
- オフラインや未ログインでもローカルだけで普通に使えます（同期だけ止まる）
- ※ `file://`（ファイル直接開き）ではGoogleログイン不可 → **公開URL https://mwmitsu.github.io/task/** で使ってください

### Windows版とのデータのやり取り（ファイル経由）
クラウド同期とは別に、**Windows常駐版との橋渡し**はファイル（CSV/JSON）で行います。

| やりたいこと | 手順 |
|---|---|
| **Windows版 → iPad** | Windows版でCSV書き出し（または `widget-data.json` をそのまま）→ クラウド/メールでiPadへ → Web版で「📥 取り込み」 |
| **iPad → Windows版** | Web版で「📤 CSV書き出し」→ そのCSVをWindows版のトレイ「CSVから取り込み」で読込 |
| **バックアップ** | Web版「📤 JSON書き出し」で全体を保存 |

- CSVの形式（列・タグの区切り・文字コードUTF-8 BOM）はWindows版と**互換**にしてあります
- 取り込みは「**本文が同じタスクは重複として除外**」して追加します

## 仕組み（技術メモ）
- 本体はHTML+CSS+JSの単一ファイル（`index.html`）。フレームワーク不要。`localStorage` に保存（キー `taskweb-data-v1`）
- PWA用に同じフォルダへ `manifest.webmanifest`・`sw.js`（Service Worker）・`icon-180.png`・`icon-512.png` を同梱
  - Service Workerは**同一オリジンのアプリ本体だけ**をキャッシュ（オフライン起動用）。Firebase等の別オリジン通信は一切触らず同期を壊さない設計
  - キャッシュを更新したいときは `sw.js` の `CACHE` 名（例 `taskweb-v2`）を上げる
- CSVは Windows版 `Export-Tasks-Csv` と同じ列順（text,done,cat,priority,due,estimate_min,tags,note,**repeat,subs**）・タグは `|` 区切り・先頭にBOM。引用符内の改行も保持。**繰り返し**はラベル（なし/毎日/平日/毎週）、**サブタスク**はタブ区切り＋各項目の先頭1文字が完了フラグ（1=完了/0=未、例 `1牛乳⇥0パン`）で本文に `|` や `✓` が含まれても壊れません — Windows版と相互運用可。旧列だけのCSVも取り込み可
- JSON取り込みは Web版の書き出しと Windows版 `widget-data.json` の両方を受け付け（分類はキー/ラベルどちらも解釈）。`hideDone`（完了を隠す）も往復
- ローカル確認用サーバ：`powershell -ExecutionPolicy Bypass -File serve.ps1 -Port 8190`（iPadで使うだけなら不要）
- デプロイ：このフォルダが GitHub リポジトリ `MWmitsu/task` のソース。`git add -A && git commit && git push` で https://mwmitsu.github.io/task/ に反映（Pagesビルド後、CDNの404キャッシュが数分残ることがある）
